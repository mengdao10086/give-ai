# 飞智散热器 B6X — 蓝牙扫描生命周期 Bug 修复历程

## 目标
修复 `com.flydigi.waspwing.experimental` 在 Android 16 (Redmi K60) 上蓝牙扫描永不停止、GATT 连接不完整的问题。

## 方案选择
LSPosed 1.9.2-it 模块（Java），不修改 APK。

## 问题链（由浅入深）

### 第 1 层：扫描永不停止（UI 显示"扫描中"）
- **现象**：ACL_CONNECTED 后扫描界面持续显示
- **诊断**：`onDeviceConnected()` 未被《蓝牙扫描生命周期 Bug 分析报告》认为未调用 stopScan()，但反编译显示 smali 已调。真正的 UI 原因是 `_waspWingInfo` LiveData 从未收到 `connected=true`。
- **修复 #1**：钩 `AbstractBluetoothController.onDeviceConnected()` → `stopScan()`
- **修复 #2**：钩 `BluetoothViewModel.onDeviceConnected()` → 更新 `_connectLiveData / _waspWingInfo`

### 第 2 层：_waspWingInfo 为 null，界面仍不可操作
- **现象**：UI 显示"已连接"但所有控制按钮灰化
- **诊断**：`_waspWingInfo` 收到的 `convertFromDevice` 对象只有 `connected=true`，但 `windLevel=0`、`coldLevel=0`、`runMode=0` 等字段都是默认值，DataBinding 因此禁用控制按钮。
- **修复尝试**：手动调用 `connectGattWith()` 触发 GATT 连接 → 但 SDK 已自动连接，造成双 GATT → 回滚。
- **最终修复 #4**：见下文。

### 第 3 层：GATT 连接成功但 onServicesDiscovered 不触发
- **现象**：`onGattConnected` 触发，`services=0`，`onServicesDiscovered` 永不回调。
- **诊断经过**：
  1. 钩 `BluetoothGatt.discoverServices()` → **日志显示该方法从未被调用**
  2. 跟踪调用链：`onGattConnected()` → `checkBluetoothPermission()` → **返回 false** → 提前 return → 不执行 `gatt.discoverServices()`
- **根因**：Android 16 (SDK 36) 上 `PermissionChecker.checkSelfPermission(BLUETOOTH_CONNECT)` 返回 false，但原应用未正确处理此情况。
   - `connect()` 中同样的权限检查能通过（原因尚不完全清楚，可能与调用线程有关）
   - 但 `onGattConnected()` 中的权限检查在 BLE 回调线程上失败
- **修复 #4**：钩 `checkBluetoothPermission()` → 强制返回 true

### 第 4 层：智能温控 UI 闪烁
- **现象**：设为智能温控模式时，模式指示器持续闪烁。切换为固定功率再切回即正常。
- **诊断**：
  1. 日志异常长：`onCharacteristicChanged` 约每秒触发一次
  2. 每次 → `App.onDeviceInfoUpdate()` → `_waspWingInfo.postValue()` → DataBinding 重渲染
  3. 但 flash 的根因是 fix #2 用 `convertFromDevice(device)` 创建了全默认值的 WaspWingInfo
  4. 默认值中 `experimentalRunModeValue=null`
  5. `App.onDeviceInfoUpdate()` 检测到 null → 调用 `setExperimentalRunMode(true, coldLevel+1)`
  6. 设备响应此命令 → 又触发数据更新 → 循环
- **修复**：去掉 fix #2 中的 `convertFromDevice`。让正常 GATT 数据流填充 (fix #4 已保障数据可达)

## 关键技术细节

| 组件 | 作用 |
|------|------|
| `AbstractBluetoothController$mConnectStateReceiver$1` | 接收 ACL_CONNECTED 等蓝牙广播，已有 stopScan() |
| `BluetoothViewModel` | 抽象 ViewModel，`onDeviceConnected()` 含 LiveData 更新逻辑 |
| `WaspWingViewModel` (app) | 具体 ViewModel，实现 `WaspWingGattCallback`，`onDeviceInfoUpdate()` 含智能温控自修复逻辑 |
| `LeDataInteractionController` | BLE 核心控制器，`onGattConnected()` 含权限检查 |
| `WaspWingDataInteractionController` | 散热器 BLE 实现，`onGattConnected()` 创建 `deviceInfo` |
| `checkBluetoothPermission()` | SDK 内部权限检查，API 31+ 查 BLUETOOTH_CONNECT，API 30- 查 BLUETOOTH |
