# tempctrl 智能温控守护程序 — 设计说明

对应实现：[tempctrl.c](tempctrl.c)
配置文件：[magisk模块框架/profile.conf](magisk模块框架/profile.conf)
本文件只说明运行逻辑，不追求代码级精细。

---

## 整体架构

```
 ┌──────────────────┐     FIFO      ┌──────────────────┐
 │ LSPosed 模块     │  '1'/'0'      │ tempctrl 守护进程 │
 │ (MainHook.java)  │ ───────────→  │ (本 C 程序)      │
 │                  │               │                   │
 │ BLE 连接 → 写 '1'│               │ 阻塞读 FIFO       │
 │ BLE 断连 → 写 '0'│               │ 收到 '1' → 开始控制 │
 └──────────────────┘               │ 收到 '0' → 休眠   │
            ↑                       │                   │
            │ am broadcast          │ 每 5s main_loop() │
            │ (控制参数)             │                   │
            ↓                       └──────────────────┘
 ┌──────────────────┐        ┌──────────────────────────┐
 │ 飞智散热器 App   │        │ profile.conf（配置文件）  │
 │ (WaspWing)      │        │ 启动时读取，运行中热重载   │
 └──────────────────┘        └──────────────────────────┘
```

## 配置文件系统 (profile.conf)

tempctrl 支持运行时配置文件，所有关键阈值和参数均可外部修改，无需重新编译。

### 配置路径自动检测

启动时通过 `/proc/self/exe` 获取自身路径，按以下顺序查找 `profile.conf`：

1. **同目录**（独立 Magisk 模块）：`/data/adb/modules/b6x-Enhancement-Plan/profile.conf`
2. **父目录**（scene_systemless/others 场景）：`/data/adb/modules/.../others/../profile.conf`
3. 都找不到 → 全部使用代码内默认值，仅记一条日志

也支持 `--config` 参数手动指定：
```shell
tempctrl --config /自定义路径/profile.conf
```

### 配置格式

```
# 注释以 # 开头
KEY=VALUE

# 例：
BATT_BASELINE=350
BATT_ZONE_1=7
```

不认识的 key 静默跳过；每个值有范围保护，非法值不会导致进程崩溃。

### 热重载

每轮 `main_loop()`（5 秒间隔）开头 `stat()` 检查配置文件的修改时间（mtime），变了就自动重读所有参数并记日志。**修改配置无需重启 daemon**。

### 可配置参数一览

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `BATT_BASELINE` | 350 | 电池基准温度 (0.1°C) |
| `BATT_ZONE_1` | 7 | 第一区间阈值 (0.1°C)，偏差在此以内不调整 |
| `BATT_ZONE_2` | 20 | 第二区间阈值 (0.1°C)，超过此值 ±2 档 |
| `CPU_EMERG_1/2/3` | 650/750/850 | CPU 紧急触发温度 (0.1°C) |
| `CPU_RECOVER_0/1/2` | 550/650/750 | CPU 紧急恢复温度 (0.1°C) |
| `EMERG_FORCED_1/2/3` | 5/7/9 | 各紧急等级强制最低档位 |
| `COOLDOWN_CYCLES` | 4 | 调整后冷却周期数 (×5秒) |
| `DISCONNECT_RESET_SEC` | 60 | BLE 断联超时重置秒数 |
| `CPU_FILTER_ALPHA` | 25 | CPU 滤波系数 (百分比) |
| `LOG_FILE` | /cache/<二进制名>.log | 日志文件路径 |

## 温度读取

- **电池温度**：`/sys/class/power_supply/battery/temp`
  - 原生 0.1°C 精度，如 350 = 35.0°C

- **CPU 温度**：`/sys/class/thermal/thermal_zone[30~40]/temp`
  - 原始值 m°C，÷100 转 0.1°C，如 75321 → 753 (=75.3°C)
  - 取多个 zone 中的最高值
  - 一阶低通滤波平滑（系数由 `CPU_FILTER_ALPHA` 配置，默认 α=0.25）

## 档位系统

档位 0~10，使用查表法（非公式计算）：

| 档位 | 模式 | 目标温 | 风扇RPM | 制冷片 | 说明 |
|-----|------|-------|--------|-------|------|
| 10 | 固定功率 | — | 6000 | 190 | 制冷峰值 |
| 9 | 智能温控 | 12°C | 5800 | auto | 紧急3-min |
| 8 | 智能温控 | 13°C | 5500 | auto | |
| 7 | 智能温控 | 14°C | 5100 | auto | 紧急2-min |
| 6 | 智能温控 | 15°C | 4600 | auto | |
| 5 | 智能温控 | 16°C | 4000 | auto | 紧急1-min |
| 4 | 智能温控 | 18°C | 3400 | auto | |
| 3 | 固定功率 | — | 2900 | 80 | 低功耗1 |
| 2 | 固定功率 | — | 2500 | 60 | 低功耗2 |
| 1 | 固定功率 | — | 2200 | 40 | 低功耗3 |
| 0 | 固定功率 | — | 2000 | 20 | 伪待机 |

控制指令通过 `am broadcast` 发送：

```shell
adb shell am broadcast --user 0 \
  -a com.flydigi.SET_TEMPERATURE \
  --ei mode <0/1> \
  --ei temperature <目标温度> \
  --ei windOC <固定转速> \
  --ei coldOC <制冷强度> \
  --ei windLevel <智能转速上限>
```

- **智能模式 (mode=0)**：设目标温度和风扇上限，散热器自行调节制冷
- **固定功率 (mode=1)**：直接指定风扇转速和制冷片强度

## 主循环逻辑

`main_loop()` 每 **5 秒**执行一次：

0. **检查配置更新** → `stat()` 检测 `profile.conf` 修改时间，变了就热重载
1. **紧急干预** → 读 CPU 温度 → 低通滤波 → 判定紧急等级 (0~3)
   - 等级与阈值可配置（默认 >65°C/75°C/85°C）
   - 降温有 10°C 滞回，逐级下降
2. **电池控制** → 读电池温度 → 与基准温度比较：
   - 偏差 ≤ 第一阈值 → **不变**（死区）
   - 第一阈值 < 偏差 ≤ 第二阈值 → **±1 档**
   - 偏差 > 第二阈值 → **±2 档**
   - 限制：温度未变化时不调、冷却期内不调
3. **合最终档位** = max(电池档位, 紧急强制最低档位)
4. **下发**控制参数（去重：参数完全相同时跳过）
5. 每周期末尾非阻塞 peek FIFO → 有 `'0'` 则进入休眠

## FIFO 状态机

```
 ┌──────────┐    peek 无 '1'     ┌───────────┐
 │  启动     │ ───────────────→  │ FIFO 阻塞  │
 │          │                    │ 等待信号   │
 │ peek FIFO│ ←───────────────  │           │
 │ 有 '1'?  │  收到 '1'进入工作  │ CPU 0%    │
 └────┬─────┘                    └───────────┘
      │                                ↑
      ▼ 有 '1'                         │ 收到 '0'
 ┌──────────┐    每 5s main_loop()    │
 │ 工作模式  │ ────────────────────   │
 │          │    peek 到 '0' ────────  │
 └──────────┘
```

**断联超时**：收到 `'0'` 后计时，下次收到 `'1'` 时如果断联 ≥ `DISCONNECT_RESET_SEC` 秒，执行 `reset_state()` 重置所有状态重新开始。

## 电池控制细节

- **基准温度**：35.0°C（`BATT_BASELINE=350`，可配置）
- **三级策略**（diff = 当前温度 - 基准温度）：
  - `|diff| ≤ BATT_ZONE_1` → 不动，属于正常范围
  - `BATT_ZONE_1 < |diff| ≤ BATT_ZONE_2` → 升/降 1 档
  - `|diff| > BATT_ZONE_2` → 升/降 2 档

- **防重复**：
  - 记录上次调整时的电池温度（`prev_batt_temp`）
  - 如果当前温度与该值相同，跳过本次升降档
  - 死区内也更新该记录，避免反复进入计算

- **冷却期**：
  - 每次调整后冻结 `COOLDOWN_CYCLES` 个周期（默认 4 周期 ≈ 20 秒）
  - 冷却期内仍读取温度并检测变化，但不执行升降档
  - 冷却期结束后如果温度仍偏离基准，立即响应

## CPU 紧急干预细节

- 使用一阶低通滤波：`weighted = raw × CPU_FILTER_ALPHA% + old × (100-CPU_FILTER_ALPHA)%`
- 首次运行直接赋值，不做滤波
- 滤波系数默认 25 (α=0.25)，可通过 `CPU_FILTER_ALPHA` 配置

**紧急等级**（阈值均可通过 profile.conf 调整）：

| 等级 | 触发条件（默认） | 强制最低档位（默认） |
|------|----------------|-------------------|
| 0 | 正常 | 无 |
| 1 | CPU > 65°C (`CPU_EMERG_1`) | 5 (`EMERG_FORCED_1`) |
| 2 | CPU > 75°C (`CPU_EMERG_2`) | 7 (`EMERG_FORCED_2`) |
| 3 | CPU > 85°C (`CPU_EMERG_3`) | 9 (`EMERG_FORCED_3`) |

**退出紧急时**限制电池档位上限，防止突然降速导致温度反弹：
- 退出等级 3 → `battery_fan_level ≤ EMERG_FORCED_3`
- 退出等级 2 → `battery_fan_level ≤ EMERG_FORCED_2`
- 退出等级 1 → `battery_fan_level ≤ EMERG_FORCED_1`

## 日志系统

- **默认日志路径**：`/cache/<二进制名>.log`（自动根据二进制文件名生成）
  - 例：`tempctrl` → `/cache/tempctrl.log`
- 可通过 `profile.conf` 的 `LOG_FILE` 覆盖
- 格式：`[时间戳] 日志内容`

## 启动流程

1. 设定默认日志路径（根据二进制文件名）
2. 加载配置（`--config` 参数 > 自动检测 > 全部默认值）
3. 创建 FIFO（删除旧的重新创建）
4. 读电池温度计算初始档位
5. 非阻塞 peek FIFO：
   - 有 `'1'` → 模块已活跃，直接进工作模式
   - 无信号 → 阻塞等待 FIFO 信号
6. 进入工作模式后每 5 秒执行 `main_loop()`

## 注意事项

- **C 代码需要前向声明**（NDK clang 严格检查）：`reset_state()` 被 `wait_for_fifo()` 调用，必须提前声明
- **FIFO 路径** `/data/local/tmp/b6x_fifo`，权限 0666，app 进程（非 root）也能写入
- **CI 编译**使用 NDK r27c，静态链接 aarch64-linux-android21，编译后需要 `python3 patch_tls.py` 修复 PT_TLS 对齐
- **温度传感器路径** `thermal_zone30~40` 是 K60 机型值，其他 SoC 可能需要调整扫描范围
- **去重逻辑**：`apply_level()` 缓存上次参数，连续相同不下发，减少不必要的 `am broadcast` 调用
- **FIFO 断连**：写端全关时 `read()` 返回 0，必须 `close()` 后重新 `open()`
- **`patch_tls.py`**：无 TLS 段时返回非零，需要 `|| echo "skipped"` 容忍
- **`#include <stdarg.h>`**：`write_log()` 用了 `va_list`，NDK clang 下不是隐式包含的
